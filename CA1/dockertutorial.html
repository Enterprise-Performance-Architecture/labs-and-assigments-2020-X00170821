Docker Tutorial

Prerequisites:

• Google Cloud Voucher

• Github account setup

Creating a Docker image

To begin, you will need to create a directory using the mkdir command

For this example, we will use

mkdir dockertutorial

Then we will create a file that tells Docker the specifications of the image you wish to create, this will be called dockerfile and we will be using nano which is a text editor.
The code below will bring up the text editor where the user can specify what they want in their image:

nano dockerfile

When you are on the text editor, we will use these options:

FROM ubuntu:latest

COPY script.sh /

RUN chmod u+x /script.sh

CMD ["/script.sh"]

What do the commands do?

FROM specifies the [image:version]

COPY will allow you to add files from your current docker directory

RUN will build the file

CMD will specify what commands the container will run. In this instance, it is pulling the commands from script.sh

Creating script.sh

If you attempt to build the docker file, you will notice that you will not be able to and will encounter, because you will still need to create script.sh as we specified that there will be commands run from the script within this file.

We can achieve this by using nano and naming it script.sh

In this instance, we will be making a CPU counting script using bash with the code below:

#!/bin/bash

cpucount=$(grep "rocessor" /proc/cpuinfo | wc -l)
echo "CPU count: $cpucount"

Building the Docker image

To build a Docker image, you will be using the docker build command.

An alternative to docker build on its own is adding the -t option to give the build a name like below

docker build -t cpucounter

Google Cloud Shell will process this for a moment and your Docker image is created!

To verify this, type in docker images to bring up a list of all available images.

CA Methodology

  1. I created a new directory EPA-CA1

  2. Created the docker file through nano dockertutorial with the commands based off the spec:
    1. FROM python:3.9

      Selects Python 3. I wondered if I should have put "latest" instead of 3.9, but if there was a 4.X version, this would go against spec

    2. WORKDIR /EPA-CA1/

      Specifies the work directory

    3. COPY dockertutorial.html /EPA-CA1/

      Copies the html file that I have made to the container's work directory in step 2

    4. CMD python3 -m http.server 8080:8080

      Python 3 server module on port 8080

  3. Created a rudimentary HTML file with what was requested in the spec
    1. Coverage of course so far in the form of a docker tutorial

    2. CA Methodology

    3. 4 links to relevant resources

  4. Built the Dockerfile using -t dockertutorial

  5. Ran the Dockerfile using docker run with the following options
    - --rm to keep the host clean from stopped and unused containers
    - -it to start the container in interactive mode
    - -p 8080:8080 to specify port

  6. Use web preview to view the directory and subsequently the HTML file

  7. Commit to Github
